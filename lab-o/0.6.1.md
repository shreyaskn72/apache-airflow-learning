Adding **Flower UI** is a smart move 👍
It gives you real-time visibility into Celery workers used by Apache Airflow.

---

# 🌸 What is Flower?

Flower is a web-based dashboard to monitor:

* Tasks (success/failure)
* Worker status
* Task duration
* Queues and retries

---

# 🛠️ Step-by-Step: Add Flower to Airflow Docker Setup

## 📌 Step 1: Edit `docker-compose.yaml`

Open your `docker-compose.yaml` and **add this service**:

```yaml
  flower:
    image: apache/airflow:2.8.1
    command: celery flower
    environment:
      &airflow-common-env
      AIRFLOW__CORE__EXECUTOR: CeleryExecutor
    ports:
      - "5555:5555"
    depends_on:
      - redis
      - postgres
```

---

## 📌 Step 2: (Recommended) Reuse Common Config

If your file already has `x-airflow-common`, use this cleaner version:

```yaml
  flower:
    <<: *airflow-common
    command: celery flower
    ports:
      - "5555:5555"
```

---

## 🔄 Step 3: Restart Services

```bash
docker compose down
docker compose up -d
```

---

## 🌐 Step 4: Access Flower UI

Open:

```
http://localhost:5555
```

---

# 👀 What You’ll See in Flower

* ✅ Active tasks

* ✅ Queue status

# 🔐 (Optional but Recommended) Add Authentication

Update Flower service:

```yaml
environment:
  FLOWER_BASIC_AUTH: admin:admin
```

Now login required:

```
Username: admin
Password: admin
```

# ⚠️ Common Issues

### ❌ Flower not loading

Check logs:

```bash
docker compose logs flower
```

---

### ❌ Port already used

Change:

```yaml
"5555:5555" → "5556:5555"
```

---

### ❌ No data showing

Make sure:

* Airflow is using **CeleryExecutor**
* Scheduler & workers are running
