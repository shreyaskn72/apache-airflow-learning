Docker is the **cleanest and fastest way** to run Apache Airflow on Windows 11 👍


---

# 🐳 Step 0: Install Docker Desktop

Install **Docker Desktop**
👉 Make sure:

* WSL2 backend is enabled
* Ubuntu integration is ON

Verify:

```bash
docker --version
docker compose version
```

---

# 📁 Step 1: Create Airflow Project Folder

```bash
mkdir airflow-docker
cd airflow-docker
```

---

# 📥 Step 2: Download Official Airflow Docker Setup

```bash
curl -LfO https://airflow.apache.org/docs/apache-airflow/stable/docker-compose.yaml
```

This is the **official file** from Apache Software Foundation.

---

# ⚙️ Step 3: Set Environment Variables

Create a `.env` file:

```bash
AIRFLOW_UID=50000
```

👉 On Windows (WSL), run this to get your UID:

```bash
id -u
```

---

# 🚀 Step 4: Initialize Airflow

Run this **one-time setup**:

```bash
docker compose up airflow-init
```

This will:

* Setup database (PostgreSQL)
* Create default user
* Initialize configs

---

# ▶️ Step 5: Start Airflow

```bash
docker compose up
```

💡 This starts multiple services:

* webserver
* scheduler
* postgres
* redis
* worker (Celery executor)

---

# 🌐 Step 6: Open Airflow UI

Go to:

```
http://localhost:8080
```

### Default login:

```
Username: airflow
Password: airflow
```

---

# 🧠 What You Just Set Up (Important)

This is NOT a basic setup—you now have:

* Celery Executor (distributed)
* Redis (queue)
* PostgreSQL (database)

👉 This is **production-style architecture**, not just learning mode.

---

# 📂 Step 7: DAGs Folder

Your folder structure will include:

```
airflow-docker/
 ├── dags/
 ├── logs/
 ├── plugins/
 ├── docker-compose.yaml
```

👉 Add your DAG files inside:

```
dags/
```

Example DAG:

```python
from airflow import DAG
from airflow.operators.python import PythonOperator
from datetime import datetime

def hello():
    print("Hello Airflow!")

with DAG(
    dag_id="my_first_dag",
    start_date=datetime(2024, 1, 1),
    schedule="@daily",
    catchup=False
) as dag:

    task = PythonOperator(
        task_id="hello_task",
        python_callable=hello
    )
```


# ⚠️ Common Issues (Very Important)

### 1. Port already in use

Change port in `docker-compose.yaml`:

```
8080 → 8081
```

---

### 2. Docker RAM issue

Airflow needs at least:
👉 **4 GB RAM (minimum)**

Set in Docker Desktop settings.

---

### 3. Slow performance

Enable:

* WSL2 backend
* Use Linux filesystem (not C drive)
